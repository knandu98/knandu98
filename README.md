## 🔬 What I Work On

My projects tend to fall into three areas:

### Systems & Data Structures
I dig into low-level performance questions — things like how memory hierarchy decisions ripple through index structures. My 2025 research project modeled **hybrid NVM-DRAM B-trees**, benchmarking a fully-persistent NVM-optimized B+-tree against an FP-Tree (selective persistence) to quantify write amplification. Built in C++17 with a custom NVM cost simulator, correctness tests, and CSV profiling output.

→ [`benchmarking_B-trees`](https://github.com/knandu98/benchmarking_B-trees)

### Data Engineering & Lakehouse Formats
I explore the evolving landscape of open table formats. An early exploration project put **DuckLake, Apache Iceberg, and Delta Lake** head-to-head on the same analytics workload — same Gapminder dataset, same query patterns, equivalent Dash apps — to surface real differences in setup cost, query latency, append performance, and metadata footprint.

→ [`lakehouse_comparison`](https://github.com/knandu98/lakehouse_comparison)

### Data Mining & Machine Learning
I'm interested in algorithms that surface non-obvious patterns. A 2024 project reimplemented **unexpected association rule mining** (NEclatClosed-style closed itemset mining + contradiction detection) in pure Python, faithfully reproducing the methodology of a Spark-based research paper at a scale you can run and demo locally in seconds.

→ [`unexpected_mining`](https://github.com/knandu98/unexpected_mining)
